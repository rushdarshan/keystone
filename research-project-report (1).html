Self-Supervised ViT (DINO), then Distilled + Quantized Edge Encoder

Problem & contribution

Labels are expensive; SSL learns strong features without them. But SSL models are large. Open question you answer: how much can a self-supervised encoder be distilled + int8-quantized before its label-free representations degrade? You reproduce DINO, then contribute the compression study the original paper did not run at this scale.

Math you'll use

  • • Cross-entropy over softened distributions; temperature sharpening
  • • Centering + EMA teacher (stop-grad, momentum update)
  • • Multi-crop augmentation, cosine schedules
  • • Post-training / QAT quantization (affine int8 mapping)
  • • kNN & linear-probe evaluation (logistic regression)

Pipeline (4 stages)

  1. 1. Reproduce DINO. Train ViT-Small/backbone with multi-crop + EMA teacher; match reported kNN accuracy trend on the chosen dataset.
  2. 2. Distill. Freeze SSL teacher, train a tiny student to mimic embeddings (feature-matching + relational KD).
  3. 3. Quantize. Apply PTQ then QAT to int8; export for CPU inference.
  4. 4. Evaluate. Linear probe + kNN + latency/size on each variant; plot the trade-off curve.

Ablation studies

  • • Distillation loss: feature-match vs. relational (RKD) vs. logit KD
  • • Student size sweep (depth × width) vs. accuracy retained
  • • PTQ vs. QAT; int8 vs. fp16
  • • # multi-crops & teacher temperature sensitivity
  • • Does the compressed model keep DINO's object-segmentation attention?

Risks & mitigations

  • • SSL is compute-hungry → use small dataset + small ViT, fewer epochs.
  • • Unstable teacher EMA → warmup + gradient clipping, follow paper schedule.
  • • Quant accuracy drop → fall back PTQ→QAT; report honestly (it's a finding).
